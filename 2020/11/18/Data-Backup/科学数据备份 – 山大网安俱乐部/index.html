思考

下面情况是否会对你的工作造成毁灭性影响？

  • 你的女朋友删除了你的论文
  • 第二天要进行课堂展示，熬夜做完了PPT，关机前不小心打翻了咖啡，液体渗入笔记本，笔记本开不了机了
  • D盘不小心被格式化了
  • 电脑中了勒索病毒，所有文件都被加密，对方索要8个比特币
  • 笔记本放在自习室，回来时已经不见了
  • U盘、外置硬盘和笔记本放在同一个包里，包在操场上丢了
  • 手机也丢了
  • 装系统忘记备份数据

上面这些突如其来的**是否会影响到你工作的正常进行

数据备份遇到的问题

  • 占用太多空间：视频啥的太占空间
  • 备份速度太慢：机械硬盘速度太慢、网盘更慢
  • 不够灵活，需要人工进行复杂操作：大部分备份方法需要定期手工操作
  • 大量小文件：代码等文件数量多，但是每一个都非常小，管理起来不方便

个人NAS

NAS（Network Attached Storage：网络附属存储）又被称为网络存储器，是基于网络的提供数据存储功能的设备

我们个人使用的都是家庭NAS或者个人NAS，

NAS支持多种协议，包括HTTP、FTP、NFS、CIFS（SMB）等

可以将NAS中的某个文件夹或者分区挂载成本地电脑的一个网络硬盘，使用体验就像使用本地硬盘一样方便

云盘

国内有百度网盘、阿里网盘、坚果云、蓝奏云、115网盘，国外有Dropbox、Onedrive、Google drive、Mega…

NAS还是云？

网盘和NAS都是提供基于网络的存储服务，到底应该选择哪个？

可靠性

虽然我更加喜欢NAS，不过不得不承认，从理论上讲用云盘比自己家里放个NAS可靠性强

将数据存在云存储服务中，首先最重要的数据不要丢失。专业云存储服务的服务器都是由专门的运维人员负责维护，他们有科学的备份方案、实时的硬盘健康度监控、会定期更换损坏硬盘，从这一方面来讲数据比存放在自己的硬盘中更加可靠。

但是我们还要考虑一些其他的因素，因为审查的存在，我们的某些文件可能会被审查系统自动删除，特别是某些涉及到版权的内容。

隐私

把自己的数据存放在云存储服务中，还有一点不得不考虑，那便是隐私

个人私密内容被泄露，相信大家都听说过什么明星艳照门，一些私密的照片存放在网盘中却被泄露

我们的照片被云存储厂商拿去做人工智能模型训练的数据集

企业的机密数据被窃取，商场如战场，说不定云存储厂商也会参与其中

而将数据存储在自己家里的NAS中，只要做好必要的安全措施（防火墙、强密码），就几乎不需要考虑隐私被泄露，毕竟数据都存在自己家里的硬盘中

文件备份的尺度

根据需要备份的文件大小和范围，可以将备份按照下面的级别分类，按照不同大小的尺度进行备份，无论是在操作复杂性上还是备份速度上都有却别

  • 行级别
  • 文件级别
  • 文件夹级别
  • 分区级别：与文件夹差不多，只是范围更大一点，不过有一些更好的办法
  • 硬盘级别
  • 字节级别

行级别

类似于git和协作文档那种，所有的修改是按照行记录的

文件和文件夹级别

最常见的，针对单个文件或者整个文件夹进行操作

分区级别

有一些现代的文件系统提供高效好用的备份方案，速度快操作简单体验好

硬盘级别

组Raid

字节级别

Dropbox等著名网盘厂商可以按照字节级别的修改进行备份

比如修改一个1G的文件，即使只改了一个字节，但百度网盘需要重新上传1个G，而具有字节级备份能力的网盘只需要上传修改的那一个字节就够了

国内厂商目前还没听说有支持字节级备份的，桑心

分级备份

  • 冷数据：CD\DVD （存储时间长）
  • 热数据: 硬盘

多备

  • 异地多备
  • 本地多备

RAID

RAID（Redundant Array of Independent Disks）独立硬盘冗余阵列，旧称廉价磁盘冗余阵列（Redundant Array of Inexpensive Disks），简称磁盘阵列

利用虚拟化存储技术把多个硬盘组合起来，成为一个或多个硬盘阵列组，目的为提升性能或资料冗余，或是两者同时提升

在运作中，取决于 RAID 层级不同，资料会以多种模式分散于各个硬盘，RAID 层级的命名会以 RAID 开头并带数字

  • RAID 0 (不具备备份能力)
  • RAID 1
  • RAID 5
  • RAID 6
  • RAID 7
  • RAID 01
  • RAID 10
  • RAID 50
  • RAID 60

每种等级都有其理论上的优缺点，不同的等级在两个目标间获取平衡，分别是增加资料可靠性以及增加存储器（群）读写性能.简单来说，RAID把多个硬盘组合成为一个逻辑硬盘，因此，操作系统只会把它当作一个实体硬盘

RAID常被用在服务器电脑上，并且常使用完全相同的硬盘作为组合。由于硬盘价格的不断下降与RAID功能更加有效地与主板集成，它也成为普通用户的一个选择，特别是需要大容量存储空间的工作，如：视频与音频制作

RAID按照实现方式不同，分为软件RAID和硬件RAID。软件RAID灵活性更好一点，并且提供完善的健康性监控。硬件RAID性能高，速度快。

RAID 1

两组以上的N个磁盘相互作镜像，在一些多线程操作系统中能有很好的读取速度，理论上读取速度等于硬盘数量的倍数，与RAID 0相同。另外写入速度有微小的降低。只要一个磁盘正常即可维持运作，可靠性最高。其原理为在主硬盘上存放数据的同时也在镜像硬盘上写一样的数据。当主硬盘（物理）损坏时，镜像硬盘则代替主硬盘的工作。因为有镜像硬盘做数据备份，所以RAID 1的数据安全性在所有的RAID级别上来说是最好的。但无论用多少磁盘做RAID 1，仅算一个磁盘的容量，是所有RAID中磁盘利用率最低的一个级别。

如果用两个不同大小的磁盘建RAID 1，可用空间为较小的那个磁盘，较大的磁盘多出来的空间也可以分割成一个区来使用，不会造成浪费。

RAID 5

RAID Level 5是一种储存性能、数据安全和存储成本兼顾的存储解决方案。它使用的是Disk Striping（硬盘分割）技术。

RAID 5至少需要三个硬盘，RAID 5不是对存储的数据进行备份，而是把数据和相对应的奇偶校验信息存储到组成RAID5的各个磁盘上，并且奇偶校验信息和相对应的数据分别存储于不同的磁盘上。当RAID5的一个磁盘数据发生损坏后，可以利用剩下的数据和相应的奇偶校验信息去恢复被损坏的数据。RAID 5可以理解为是RAID 0和RAID 1的折衷方案。RAID 5可以为系统提供数据安全保障，但保障程度要比镜像低而磁盘空间利用率要比镜像高。RAID 5具有和RAID 0相近似的数据读取速度，只是因为多了一个奇偶校验信息，写入数据的速度相对单独写入一块硬盘的速度略慢，若使用“回写缓存”可以让性能改善不少。同时由于多个数据对应一个奇偶校验信息，RAID 5的磁盘空间利用率要比RAID 1高，存储成本相对较便宜。

RAID 6

与RAID 5相比，RAID 6增加第二个独立的奇偶校验信息块。两个独立的奇偶系统使用不同的算法，数据的可靠性非常高，任意两块磁盘同时失效时不会影响数据完整性。RAID 6需要分配给奇偶校验信息更大的磁盘空间和额外的校验计算，相对于RAID 5有更大的IO操作量和计算量，其“写性能”强烈取决于具体的实现方案，因此RAID 6通常不会通过软件方式来实现，而更可能通过硬件方式实现。

同一数组中最多容许两个磁盘损坏。更换新磁盘后，资料将会重新算出并写入新的磁盘中。

RAID 6必须具备四个以上的磁盘才能生效。可使用的容量为硬盘总数减去2的差，乘以最小容量

RAID 6在硬件磁盘阵列卡的功能中，也是最常见的磁盘阵列等级。

Dedup数据去重

文件多次备份会占用很多额外的存储空间，目前有一些技术可以提供文件级别、字节级别的去重，可以将相同的数据只保留一份，节省存储空间

数据去重原理

Windows 在Server版本中可以手工启用数据去重功能，详情见 微软官方文档

Linux系统下数据去重功能是由文件系统提供的，包括Btrfs和Zfs都提供文件去重功能

其他备份

一些容易忽略的内容，一旦丢了就后悔没有进行备份

浏览器

很多人会忽视掉浏览器数据的备份，直到重装系统后才发现自己的密码都存在浏览器里，一些常用的网站标签找不到了工作效率下降

  • 标签
  • 插件
  • 一些配置信息
  • 密码

现代浏览器一般都可以登录账号进行同步

利用工具定期导出备份，例如：moonD4rk/HackBrowserData, 别问为啥名字里有个Hack

账号和密码备份

你有备份自己的账号和密码吗？ （另外一个问题，你不会所有的网站都用的同一个邮箱同一个密码吧）

  • 1Password\LastPass\KeePass等密码管理软件
  • 浏览器自动填充？
  • Excel表格
  • 纯文本打印出来

我个人密码比较多，几百个账号都是不同的密码，所以喜欢现代的密码管理工具，它们大多提供云服务进行备份，或者提供插件将加密后的密码数据备份到你常用的网盘

笔记备份

笔记的特点：长度短、内容杂、数量多

  • 手机便签（不是所有手机都自动备份便签吧）
  • 印象笔记\Notion\OneNote等笔记软件
  • 文件存储，按照文件备份方式来

通讯录

  • 手机自带
  • 网盘
  • 纸质通讯录
  • 废弃手机卡

将通讯录备份到废弃不用的手机卡中，然后将手机卡存放在家里某个抽屉里

软件备份

  • 软件名字（我在刷机前都会截图，记录自己都安装了哪些软件。现在小米自带的云存储可以存软件了，登录账号自动就下载下来了）
  • 软件配置信息、配置文件、缓存数据（特指微信，自己不存他云里有也不给你，重装后啥聊天记录都没了）
  • 苹果平台会自动备份这些吧，体验应该超好

使用git

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。它是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。是世界上最先进的分布式版本控制系统。

可以使用 github\gitee\coding\bitbucket\gitlab 等提供的Git仓库服务来备份自己的代码

存北极

2020 年 2 月 2 日，GitHub 将捕获每个活动公共存储库的快照，并将其保存在 GitHub Arctic Code Vault 中。这些数据将存储在 3500 英尺的胶片卷轴上，该胶片卷轴由一家专门从事长期数据存储的挪威公司 Piql 提供并编码。
